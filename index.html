What are semantic elements?

And why should you use them?

With the finalization of HTML 5, some tags have been deprecated and should not be used. These include:
  • acronym
  • applet
  • basefont
  • big
  • center
  • dir
  • font
  • frame
  • frameset
  • noframes
  • strike
  • tt
In HTML, semantic elements are tags which clearly convey their purpose and usage. Semantic elements not only make web development easier, they also allow for greater readabitity. Websites using semantic tags are more easily translated to different languages, maintain a consistent appearance when viewed through different browsers, and are more accessible to adaptive technologies for the visually impaired. Semantic HTML tags also make website content more easily recognized by search engines. Non-semantic tags, by contrast, are very broad and rely on additional code to specify their precise function on the page. These include such tags as "div" and "span". Although non-semantic elements are still usable in HTML 5, they should be avoided in favor of semantic elements whenever possible.
